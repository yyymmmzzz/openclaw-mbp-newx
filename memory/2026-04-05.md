## 桔子商圈 Coze 工作流优化（infoget_test）

### 问题诊断
- 用户输入"寻找澳洲所有投资人的资源"
- 知识库返回的数据中，"对方类型=投资方"指的是**融资方**（正在寻找投资人的公司），而非**资金方**（投资机构/VC/PE）
- 原工作流只输出 index + userID，丢失所有公司详细信息

### 优化内容
1. **关闭「表格SQL查询」** — 避免强制添加"业务类型包含投资服务"过滤条件
2. **大模型_1（Query生成）提示词优化** — 智能匹配，不再强制加业务类型限制
3. **大模型（匹配节点）提示词优化** — 核心：增加资金方vs融资方区分逻辑，输出完整11字段
4. **知识库检索** — 召回9条，匹配输出4条

### 核心提示词逻辑
```
当用户找"投资人"时：
- 资金方（投资机构）：有钱、寻找项目 → 这是用户想要的
- 融资方（项目方）：需要钱、寻找投资人 → 这是用户不想要的
- 区分关键词：融资方=出现"寻求投资人"、"寻找合伙人共同投资"
```

- agency-agents 里687个Agent，没有专门做 B2B商业目录意图匹配 的
- Coze工作流 + 精心设计的提示词 是该场景的最佳解决方案
- 工作流本身（RAG + 意图分类）已验证是正确架构
